This repository contains a distributed data storage and processing system developed using HDFS, Apache Spark, PostgreSQL, and Streamlit. The project focuses on optimizing distributed data processing and comparing partitioned and non-partitioned datasets to evaluate performance improvements. The work considers the research on the problem of optimizing distributed data storage, which was carried out using the Python programming language, using the PostgreSQL database and HDFS, PySpark, Dask, Docker technologies. The research includes a theoretical part with an analysis of the literature, as well as a practical part with the design and implementation of a software system. The software part ensures data integrity, fast access and storage optimization in a distributed data system using modern Big Data technologies. The implemented solution allows you to effectively work with large-scale data using distributed computing mechanisms. In addition, the work has a data-driven assessment approach using a linear regression model, through which the health status of patients is assessed based on biological indicators. This approach shows how to integrate artificial intelligence (AI) elements into a distributed data analysis system. Thus, the research presented is a combination of theoretical and practical components aimed at ensuring effective management and analysis of distributed data.
